# Design Style Guide

Visual target: a practical desktop cockpit that borrows Command Code's own visual grammar without copying assets.

## Core motifs

- Black background
- Thin grid lines
- Large white confident type
- Small gray explanatory copy
- Purple primary action
- Blue/purple glow fields
- Monospace command pills
- Dense operator panels
- Minimal nested cards

## UI rules

- Make risky states visible: plan, auto-accept, yolo, trusted project.
- Keep terminal and controls on one surface.
- Use panels for operator decisions, not decoration.
- Prefer direct command buttons over generic menus.
- Do not hide core functionality behind hover.
- The terminal must feel primary, not embedded as an afterthought.
